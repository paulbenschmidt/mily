- 2025-09-21: Learned about sessions
    - Django sessions are created when calling the `login()` function. The backend database then stores the `_auth_user_id`, `_auth_user_backend`, and `_auth_user_hash` in the database and passes along the `sessionid` cookie to the frontend, which is a look-up key for the session in the database. This way credentials are never sent to the frontend, and browser only needs to send the `sessionid` cookie to the backend.
- 2025-09-19: Learned about CSRF and cookies
    - CSRF (Cross-Site Request Forgery) attacks exploit the trust that a website has in a user's browser. CSRF tokens must be explicitly included in requests, and malicious sites cannot access these tokens due to Same-Origin Policy restrictions - whether they're stored in cookies, HTML, or accessed via JavaScript. Same-Origin Policy restrictions are enforced by the web browser (not by the server) and are automatically enforced unless using side-stepped with a custom browser or browser extension.
    - Cookies are automatically included in the request (unlike CSRF tokens). These can be sent with every request, and they are automatically included in the request headers; however, they cannot be read by another website (due to Same-Origin Policy restrictions). In other words, SOP prevents reading responses, not requests.
    - Cookies solve the problem of authentication/session management ("Who is this user?") while CSRF tokens solve the problem of request authenticity ("Did this request really come from my application?").
    - There are two main levels of defense: the browser and the server.
        - The browser enforces the Same-Origin Policy (covered above) which prevents the browser from reading the request from another website. However, requests can still be sent from another website, which is why...
        - The server, by default, should reject/block requests that come from other websites; however, with a CORS (Cross-Origin Resource Sharing) policy, you can allow certain websites to interact with the server.
        - CSRF tokens straddle the browser and server since the browser cannot read CSRF tokens from other websites (due to Same-Origin Policy restrictions) and the server validates that the CSRF token is valid.
