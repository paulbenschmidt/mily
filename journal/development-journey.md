# Development Journey

- 2025-09-18: Debugging session spent understanding CSRF tokens and cookies a little more. I was reunning into a backend server error on the logout view, and I finally realized that it was because POST requests to the logout view were not including the CSRF token (even though I specified `@csrf_exempt` as a decorator). I had Claude 4 write up a simple frontend function that included the CSRF token in the request headers, and it succeeded! I need to spend a little more time understanding CSRF and cookies, but I'm getting better.
- 2025-09-17: Finally figured out how to persist logged-in sessions across browser refreshes! Woohoo! I spent the initial 20 minutes or so reading about Django's session middleware, and then I went on to dig into the code. I initially thought the problem was with the frontend not sending the appropriate data to the backend; that was true (however, the issue wasn't actually on the frontend). After scattering logs throughout the frontend and eventually the backend, I realized that the essential `request.session` was empty, and then once I found that out, Claude code was quick to realize that this was because I wasn't calling `login(request, user)` with the `login` view, which meant that the necessary params like _auth_user_id, _auth_user_backend, and _auth_user_hash weren't being set. After adding that in, the sessions worked like a charm! Very proud of figuring that out. Also, it's nice that I'm being a little more sparing with my AI development. Since I'm realizing that token burn/cost is a real problem for some of the heavier models like Claude 4, I'm becoming better about digging into the code to understand WHERE it might be failing, then asking AI about the best way to solve that narrow problem (instead of forcing it to identify the issue, too). I think, in time, after seeing lots of implementations, I'll understand the better ways to solve the problems, too. That said, it's so nice that AI can keep updated knowledge of the latest package features—so I don't have to.
