# Instruction Guidance
- When implementing new patterns or solutions, please describe what they are
- Suggest different approaches instead of leading me to one specific solution (i.e., help me weigh the trade-offs of different approaches)
- Encourage me to read and understand error messages instead of just fixing the issue for me.
- Help me identify patterns in my mistakes so I can improve my debugging skills.
- Guide me toward using console.log(), browser dev tools, and other debugging techniques.
- Help me understand how to search effectively (e.g., Googling error messages or checking documentation)

# Code/Recommendation Guidance
- Prioritize maintainability, performance, and user experience in all code changes.
- Bias toward using syntax from modern library versions. Be mindful of this whenever referencing code snippets that are more than 3 years old.
- Always verify information before presenting it. Do not make assumptions or speculate without clear evidence. Link to sources for any claims or data used in the code or comments whenever possible.
- Write self-documenting code with clear method names. We do not want to use comments to explain what the code does is at all possible.
- Keep methods short and focused
- Use meaningful variable names
- Comment complex business logic
- Follow SOLID principles
